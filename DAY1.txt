Python Program – Reverse a Three Digit Number
---------------------------------------------

no = int(input("Enter any value: "))

n1 = no % 10        # Get last digit
no = no // 10       # Remove last digit
n2 = no % 10        # Get middle digit
n3 = no // 10       # Get first digit

rev = n1*100 + n2*10 + n3
print(rev)

Example
Input : 123
Output: 321


------------------------------------------------
ALGORITHMS AND DATA STRUCTURES
------------------------------------------------

Algorithm
---------
An algorithm is a set of clear step-by-step instructions used to solve a problem or perform a specific task in a program.

Example:
Sorting numbers, searching an element, or calculating factorial.


Data Structures
---------------
A data structure is a way of organizing and storing data in a computer so that it can be accessed and processed efficiently.

Examples:
Arrays, Linked Lists, Stacks, Queues, Trees and Graphs.


Classification of Data Structures
---------------------------------

1. Primitive Data Structures  
These are basic data types provided by the programming language.

Examples:
Integer  
Float  
Character  
String  
Boolean  


2. Non-Primitive Data Structures  

These are complex structures built using primitive data types.

They are further divided into two categories.

A) Linear Data Structures  
In these structures, elements are arranged in a sequential order.

Examples:
Array  
Linked List  
Stack  
Queue  


B) Non-Linear Data Structures  
In these structures, elements are connected in hierarchical or network form.

Examples:
Tree  
Graph  


------------------------------------------------
BIG O NOTATION
------------------------------------------------

Big O notation is used to represent the efficiency of an algorithm as the input size grows.

It helps programmers understand how the running time or memory usage of an algorithm changes when the amount of data increases.

Example:
Comparing the performance of a program with 10 inputs versus 1,000,000 inputs.


Types of Complexity
-------------------

1. Time Complexity  
Time complexity measures how long an algorithm takes to execute.

Example:  
Code A takes 30 seconds  
Code B takes 60 seconds  

Therefore, Code A is more efficient.


2. Space Complexity  
Space complexity measures the amount of memory required by an algorithm during execution.


Performance Cases of Algorithms
-------------------------------

Best Case  
Minimum time required for execution.

Average Case  
Normal or expected running time.

Worst Case  
Maximum time required by the algorithm.


Common Big O Notations
----------------------

O(1)      Constant Time  
O(n)      Linear Time  
O(n²)     Quadratic Time  
O(log n)  Logarithmic Time  
O(2ⁿ)     Exponential Time  

Other related notations include:  
Big Omega (Ω) – Best case complexity  
Big Theta (Θ) – Average case complexity  


------------------------------------------------
RULES FOR MEASURING COMPLEXITY
------------------------------------------------

Rule 1  
A simple assignment or single statement has complexity O(1).

Example
a = 5


Rule 2  
A loop that runs from 0 to n has complexity O(n).

Example

for i in range(n):
    print(i)


Rule 3  
Nested loops usually result in quadratic complexity O(n²).

Example

for i in range(n):
    for j in range(n):
        print(i, j)


Rule 4  
If the loop variable is divided by 2 in each iteration, the complexity becomes O(log n).

Example

n = 16

while n > 1:
    print(n)
    n = n // 2


Rule 5  
When multiple statements run one after another, their complexities are added.

Example

for i in range(n):
    print(i)

for j in range(n):
    print(j)

Total complexity = O(n)


------------------------------------------------
COMMON QUESTIONS
------------------------------------------------

1. Difference Between Big O and Big Omega

Big O (O)  
Represents the worst-case complexity of an algorithm.

Big Omega (Ω)  
Represents the best-case complexity of an algorithm.


2. Difference Between Linear and Non-Linear Data Structures

Linear Data Structures  
Data elements are arranged sequentially.

Examples:
Array, Stack, Queue


Non-Linear Data Structures  
Data elements are arranged hierarchically.

Examples:
Tree, Graph


3. How O(log n) Complexity Occurs

Logarithmic complexity occurs when the problem size is reduced by half in each step.

Example Code

n = 64

while n > 1:
    print(n)
    n = n // 2

Sequence:
64 → 32 → 16 → 8 → 4 → 2 → 1

Number of steps ≈ log₂ n


4. Grid Example Complexity

If two loops iterate over height (h) and width (w):

for i in range(h):
    for j in range(w):
        print(i, j)

Time Complexity = O(h × w)
